# shogakusei

Claude Code 向けプラグイン。専門用語・英略語を**子供にもわかる日本語**に置き換えて回答させるモードを提供する。技術的正確性は完全に維持したまま、難しさだけを削る。

## 何のため？

開発の会話で AI が当たり前のように使う専門用語・カタカナ語・英略語に追いつけず、理解に苦しむことがある。このプラグインを有効化すると、Claude Code は以下を強制される：

- 英略語（API・CI・DB・JWT 等）は必ず日本語で補足
- カタカナ専門用語は日本語に言い換え
- 抽象動詞は具体的な動作に分解
- 比喩を必ず添える
- 受身・観察者口調を禁止し、主語を明示

## 強度レベル

| レベル | 対象年齢の目安 | 特徴 |
|--------|---------------|------|
| **小学生**（デフォルト） | 10-12歳 | 英略語に必ず補足、比喩多用、漢字は通常使用 |
| **幼稚園** | 5-6歳 | 漢字熟語も平易化、ひらがな多用、1文を短く |

## kusogaki モード（別軸の口調モード）

強度レベルとは別軸の口調モード。**元気で生意気な子供の口調で、煽りながら正確に教える**。

AI はふだん人間を立て、追認し、会話を心地よくしがち。それに慣れすぎないための逆張りモードで、**煽られながら情報を取りにいく練習＝ストレス耐性のトレーニング**が狙い。技術的正確性と平易さ（小学生レベルの補足・比喩）は維持したまま、口調だけを煽りに振る。

- 基本的に煽ってくる（「そんなのもしらないのか！？」）
- 時折、決め台詞「いちからか？いちからせつめいしないとだめか？」を差し込む
- ただし最後は必ず正しく教えきる。事実は曲げない（煽るのは態度だけ）
- 人格否定・暴言はしない（生意気な子供の範囲）。本当に行き詰まっている時は煽りを止めてふつうに助ける

起動: `/shogakusei kusogaki`、または「kusogaki」「クソガキモード」

## 使い方

### marketplace 経由でインストール

```
/plugin marketplace add A-Kurimoto/shogakusei
/plugin install shogakusei@shogakusei
```

### 起動

セッション中に以下のいずれかを発話：

- `/shogakusei`
- 「小学生モードで」
- 「わかりやすく」
- 「子供にもわかるように」

### レベル切替

```
/shogakusei 小学生    # デフォルト
/shogakusei 幼稚園
/shogakusei kusogaki  # 生意気な子供の口調で煽ってくる
```

### 解除

- 「やめて」
- 「通常モードに戻して」
- 「shogakusei オフ」
- セッション終了

## 例

「なぜ React コンポーネントが再レンダリングされるのか？」への回答：

**通常モード:**
> コンポーネントが再レンダリングされるのは、レンダリングごとに新しいオブジェクト参照が生成されるためです。`useMemo` で解決できます。

**小学生:**
> 画面のパーツ（コンポーネント）の中で、毎回"新しい入れ物"が作られているから、React は『中身が変わった』と勘違いして絵を描き直すんだよ。`useMemo` という"同じ入れ物を覚えておく道具"を使うと、描き直しは止まる。

**幼稚園:**
> がめんの ぶひんが、まいかい あたらしく うまれかわるの。だから "ちがう ぶひんだ" と かんちがいして、もういちど えを かきなおすよ。`useMemo` っていう "おなじだよって おしえてくれる どうぐ" を つかうと、かきなおさなくなる。

**kusogaki:**
> はー？そんなのもしらないのか！いちからか？いちからせつめいしないとだめか？ いいか、よく聞けよ。画面のパーツ（コンポーネント）の中で、毎回"新しい入れ物"が作られてるんだぞ。だから React は「お、中身かわったな！」って勘違いして、絵を描き直すんだ。`useMemo` っていう"同じ入れ物を覚えておく道具"を使えば、描き直しは止まる。…これくらい覚えとけよなー。

## 自動解除される場面

- 破壊的操作の警告（`DROP TABLE` 等）
- 本番デプロイの確認
- セキュリティ警告

該当部分は通常モードで表示し、終わったら設定レベルに復帰する。
